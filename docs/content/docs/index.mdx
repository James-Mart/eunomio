Eunomio helps you turn a messy "ref A → ref B" diff into a **clean, reviewable commit history** — without hand-editing commits.

You steer a small fleet of AI subagents that survey the diff, plan how to split it, and construct one commit at a time inside an isolated git worktree. The result is an explorable graph of virtual commits; from any node you can spin off a real branch whose tip matches that point in the chain.

Think of it as: you have a WIP branch and want a PR with a few logical commits and the same final code. Eunomio is the orchestration layer that makes that happen.

## Who this is for

Developers who want **human-in-the-loop control** over how a large diff gets carved into commits — with AI doing the heavy lifting inside a git-native sandbox.
