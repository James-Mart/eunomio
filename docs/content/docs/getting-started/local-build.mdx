Prebuilt binaries and curl install are coming later. For now, build from source.

## Prerequisites

| Tool | Notes |
| --- | --- |
| **Node.js** | 20+ (for helper + frontend build) |
| **npm** | Bundled with Node |
| **Rust** | Stable toolchain with **cargo** |
| **git** | On your `PATH` — eunomio shells out to it for all repo operations |

## Build

Clone the repository, then from the repo root:

```bash
npm install
npm run build
```

This produces `target/release/eunomio` — a single binary that serves the UI and API on one port.

## Run

Start from any directory:

```bash
/path/to/eunomio/target/release/eunomio --port 3001
```

Open `http://127.0.0.1:3001` in your browser.

### Put it on your PATH

```bash
ln -sf /path/to/eunomio/target/release/eunomio ~/.local/bin/eunomio
```

## State directory

Runtime state lives in `~/.eunomio/` by default:

- SQLite database (sessions, partitions, settings)
- Managed bare clones for network-remote sessions
- Per-partition git worktrees

This directory is shared across every repository you use eunomio with.

## Stale databases

Eunomio is pre-release; the schema changes without migration shims. If the binary fails to start on an old database, wipe state and retry:

```bash
eunomio --new
```

Also remove leftover worktrees under `~/.eunomio/worktrees/` if needed.
